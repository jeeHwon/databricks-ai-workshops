Pre-Workshop Preparation

Day Before

  • Workspace has serverless SQL compute enabled

  • Unity Catalog configured with a usable catalog
  • Vector Search enabled
  • Databricks Apps enabled
  • AI Gateway Beta enabled (optional — Section 2 can be skipped)

  • Inform participants to run 00-utils first as well

Have Ready

Section 1: Codebase & Data Setup

15 min
We're going to build an AI assistant for a fictional grocery chain called FreshMart. First, we need data — customer records, transactions, products, stores, and store policy documents. This setup notebook creates everything in about 10 minutes.
  1. Show the GitHub repo structure — point out simple/, medium/, data/
  2. Navigate to Workspace > Import from URL
  3. Open data/workspace_setup_script/01_quickstart_setup.py
  4. Fill in the Catalog Name widget, explain Schema Name
  5. Click Run All — explain what each cell does as it executes
  6. While waiting: walk through what resources will be created (tables, VS index, Genie, experiment)
While this runs, let me show you what it's creating. We'll get 6 structured tables for our retail data, a chunked policy documents table for RAG, a Vector Search endpoint and index, a pre-configured Genie Space, and an MLflow experiment for tracking.
If the setup notebook fails: check catalog/schema permissions. If the VS endpoint takes too long, proceed to Section 2 (AI Gateway) while it provisions — it only blocks Section 3.
Ask participants to raise their hand (or Slack emoji) when their notebook finishes. Use the wait time for Q&A about the architecture.

Section 2: AI Gateway (Beta)

10 min
Before we use models, let's look at governance. AI Gateway is like an API gateway specifically for LLMs — rate limiting, fallback routing, guardrails, logging. This is how you govern AI in production.
  1. Click AI Gateway in the sidebar
  2. Show the list of available routes and endpoints
  3. Demonstrate Fallback Routing — if Claude is down, route to Llama automatically
  4. Show Rate Limiting — per-user and per-team limits
  5. Show Guardrails — PII detection, content filtering rules
  6. Show Payload Logging — every prompt/response recorded
Think of this as your single pane of glass for all LLM traffic. You set it once, and every application that routes through it gets governance for free. No code changes needed in the applications themselves.
If AI Gateway is not enabled in the workspace: skip entirely. Say "This is a beta feature being rolled out — here's what it does conceptually" and show a screenshot or slide. Move to Section 3.

Section 3: Explore Capabilities (Genie, Vector Search, Playground)

30 min

3A: Genie Space (10 min)

Genie translates natural language to SQL. It understands your schema, column descriptions, and even joins across tables. Let's ask it some business questions about our FreshMart data.
  1. Click Genie in the sidebar > open "FreshMart Retail Data"
  2. Ask: "What are the top 5 products by revenue?"
  3. Show the generated SQL — point out how it joins tables correctly
  4. Ask: "Show revenue by store for the last 6 months"
  5. Show the chart visualization options
  6. Let participants try their own questions (2-3 min free exploration)

3B: Vector Search (10 min)

Vector Search is how we do semantic lookups — not keyword matching, but meaning matching. We chunked FreshMart's policy documents and embedded them. Let's search them.
  1. Navigate to Catalog Explorer > your schema > policy_docs_index
  2. Click the index > Query tab
  3. Search: "return policy for perishable items"
  4. Show the results — highlight relevance scores
  5. Try: "Do you accept EBT?" — shows it finds related info even without exact match

3C: Playground (10 min)

Now the magic — the Playground lets us combine tools into a single agent, no code. We'll attach Genie (for data) and Vector Search (for policies) and get a unified assistant.
  1. Click Playground in the sidebar
  2. Select a model (Claude Sonnet 4 or Llama 3.3 70B)
  3. Click Tools > Add Genie Space > select FreshMart
  4. Click Tools > Add Vector Search Index > select your index
  5. Test: "I bought frozen fish and it was bad. Can I return it? Also, how much revenue do we get from fish products?"
  6. Show it uses BOTH tools to answer a single question
  7. Point out the Export to App button — "This is how we go to production"
This is a great "wow moment." Ask participants to try a multi-tool question and share the best one in chat. Create a mini competition for the most creative question.
If Vector Search index is still not ONLINE: demo Genie + Playground without VS. Say "Vector Search is still syncing — you'll be able to add it in a few minutes. Let's see what Genie alone can do."

Section 4: Databricks Apps (Deploy Custom Agent)

20 min
The Playground is great for prototyping, but for production you want version control, custom logic, and a proper UI. Databricks Apps lets us deploy a Python agent with a chat interface as a managed web app.
  1. Open simple/L100-agent-openai-sdk/ in the workspace
  2. Walk through agent_server/agent.py — show MODEL, SYSTEM_PROMPT, MCP_SERVERS
  3. Explain the databricks.yml bundle config — permissions, resources, environment
  4. Update the resource identifiers (Genie Space ID, VS index name, experiment ID)
  5. Deploy: show the deploy command or trigger from the workspace UI
  6. Open the running app URL > send test messages
This is production-grade: it has a service principal for security, MLflow tracing for observability, and the Databricks Asset Bundle manages the lifecycle. You can git-push to deploy.
If deployment fails: check the error. Most common issues are (1) app name conflict — delete or bind existing, (2) missing permissions — verify service principal grants in databricks.yml. Show logs with databricks apps logs <app-name>.
Have participants customize the SYSTEM_PROMPT before deploying — give them 3 minutes to write a persona (helpful grocer, sarcastic chef, formal analyst). Seeing different personalities from the same data is memorable.

Section 5: MLflow Evals & Governance

30 min
Deploying is just the start. How do you know your agent is good? MLflow gives us tracing — every tool call, every LLM decision — plus 70+ automated judges that score quality, safety, and relevance.

5A: Traces (10 min)

  1. Open Experiments > your freshmart experiment
  2. Click on a trace — show the span tree (user message → tool calls → response)
  3. Point out latency per span, token counts, tool inputs/outputs
  4. Show how you can filter traces by status, latency, or tool used
Every conversation with your agent is automatically traced. You can see exactly which tools were called, what data came back, and how long each step took. This is observability built in.

5B: Automated Evaluation (15 min)

  1. Open simple/01_simple_agent_evaluation.ipynb
  2. Walk through the notebook structure: load traces → define scorers → run evaluation
  3. Show available LLM judges: relevance, groundedness, safety, guideline adherence
  4. Run the evaluation (uses traces already collected from app usage)
  5. Show the results table — scores per question, aggregates, failures
You don't need hand-labeled datasets to start evaluating. MLflow can judge your agent using LLM-as-judge — 70+ built-in scorers covering relevance, safety, groundedness, tone, and more.

5C: Prompt Iteration (5 min)

  1. Show how to modify the system prompt in agent.py
  2. Redeploy and generate new traces
  3. Compare evaluation scores between prompt versions
If no traces exist yet: have participants send 5-10 messages to the app first, then return to evaluation. Or use the pre-loaded traces from your instructor run.

Section 6: Human in the Loop

20 min
Automated judges are powerful, but domain experts catch things LLMs miss. Labelling sessions let your subject matter experts review agent conversations and provide structured feedback that feeds back into evaluation.
  1. In the MLflow experiment, select 5-10 traces
  2. Click Create Labelling Session
  3. Configure: add expected responses or quality rubrics
  4. Open the labelling UI — show how a reviewer rates each response
  5. Show the Review App — a shareable URL for non-technical reviewers
This closes the loop: deploy → observe traces → auto-evaluate → human review → improve prompt → redeploy. That's the full agent lifecycle, and everything stays within the Databricks platform.
Have pairs of participants review each other's agents. Person A reviews Person B's traces and vice versa. Creates engagement and surfaces interesting failure modes.
If labelling sessions UI isn't loading: ensure traces have status "OK" (not failed). You need successful traces to create a session.

Section 7: Managed Agents (Knowledge Assistant + Supervisor)

20 min
What if you don't want to manage agent code at all? Databricks has two managed agent types: Knowledge Assistant for document Q&A, and Supervisor Agent for orchestrating multiple tools/agents with zero code.

7A: Knowledge Assistant (10 min)

  1. Navigate to the KA creation UI (Machine Learning > Serving)
  2. Create a new Knowledge Assistant
  3. Point it at the Vector Search index (policy documents)
  4. Set instructions: "Answer questions about FreshMart store policies"
  5. Test it: "What is the return policy?"
  6. Show how it cites source documents

7B: Supervisor Agent (10 min)

  1. Create a Supervisor Agent
  2. Add child agents: the Knowledge Assistant + the Genie Space
  3. Set routing instructions: "Use KA for policies, Genie for data questions"
  4. Test multi-skill question: "What's our return policy, and how many returns did we process last month?"
  5. Show how the supervisor routes sub-questions to the right agent
This is the no-code path to production agents. Knowledge Assistant gives you instant RAG, Supervisor gives you multi-agent orchestration. For teams that want more control, there's the custom agent path we did in Section 4.
If managed agent creation takes too long to provision: demo from your pre-created instructor setup. Show the create flow, then switch to your working version for the live Q&A.

Common Audience Q&A

QuestionAnswer
Can I use my own model (GPT-4, Gemini)? Yes — the agent uses the Foundation Model API which supports external models via AI Gateway. You can also bring models via Model Serving endpoints.
How much does this cost? Serverless compute bills per-second. Vector Search endpoints have an hourly rate while running. Foundation Model API is pay-per-token. No idle charges for the agent app itself.
Can the agent write to tables / modify data? By default, no. Tools are read-only (Genie generates SELECT queries, VS is read-only). You'd need to explicitly add write-capable UC functions.
How do I add my own data? Replace the setup notebook's synthetic data with your real tables. Update the Genie Space to point at your tables, and load your documents into Vector Search.
Is this SOC2 / HIPAA compliant? Databricks is SOC2 Type II certified. For HIPAA, you need the Healthcare add-on. All data stays in your cloud account — no data leaves to third parties unless you configure external model endpoints.
Can I deploy this to customers (external-facing)? Yes — Databricks Apps supports custom domains and can be embedded. You'd want to add guardrails (AI Gateway), rate limiting, and thorough evaluation before going external.
What's the difference between L100 and L200? L100 (this workshop) uses managed services — Playground, managed agents, no coding. L200 goes deeper: custom agent code with OpenAI SDK, MCP servers, Lakebase for memory, Claude Code for development.
Can I use LangChain / LlamaIndex? Yes, but this workshop uses the OpenAI Agents SDK with MCP. Databricks supports any Python framework — LangChain, LlamaIndex, CrewAI — via Model Serving.
What about data freshness? Genie queries live tables (real-time). Vector Search supports continuous sync from Delta tables — new documents are indexed automatically within minutes.
How do I monitor in production? MLflow tracing for per-request observability. Lakehouse Monitoring for data quality. AI Gateway for aggregate metrics. Databricks alerts for anomaly detection.
